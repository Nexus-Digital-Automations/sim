# Versioning & History

The Sim API provides comprehensive version control for workflows, allowing you to track changes, create named versions, and restore previous states. Whether you need to rollback problematic changes or maintain stable releases, the versioning system has you covered.

## Version Management

### Automatic Versioning

Every workflow change automatically creates a new version:

- **Auto-save** - Changes saved every 30 seconds while editing
- **Manual save** - Explicit save creates new version
- **Deploy** - Deployment always creates tagged version
- **Import** - YAML/JSON imports create new version

### Version Types

| Type | Description | Retention | Auto-Created |
|------|-------------|-----------|--------------|
| **Draft** | Work-in-progress versions | 30 days | ✅ |
| **Saved** | Manually saved versions | 1 year | ❌ |
| **Tagged** | Named release versions | Permanent | ❌ |
| **Deploy** | Deployed versions | Permanent | ✅ |

## Best Practices

### 1. Version Tagging Strategy

<Callout type="info">
**Semantic Versioning**: Use semantic versioning (MAJOR.MINOR.PATCH) for consistent version management.
</Callout>

```javascript
// Implement semantic versioning
class VersionManager {
  constructor() {
    this.versionRegex = /^v(\d+)\.(\d+)\.(\d+)(?:-(.+))?$/;
  }

  parseVersion(tag) {
    const match = tag.match(this.versionRegex);
    if (!match) return null;
    
    return {
      major: parseInt(match[1]),
      minor: parseInt(match[2]),
      patch: parseInt(match[3]),
      prerelease: match[4] || null
    };
  }

  getNextVersion(currentTag, type) {
    const current = this.parseVersion(currentTag);
    if (!current) throw new Error('Invalid version format');

    switch (type) {
      case 'major':
        return `v${current.major + 1}.0.0`;
      case 'minor':
        return `v${current.major}.${current.minor + 1}.0`;
      case 'patch':
        return `v${current.major}.${current.minor}.${current.patch + 1}`;
      default:
        throw new Error('Invalid version type');
    }
  }

  async createRelease(workflowId, type, message) {
    const history = await getVersionHistory(workflowId, { limit: 1 });
    const currentTag = history.versions[0]?.tag || 'v0.0.0';
    
    const newTag = this.getNextVersion(currentTag, type);
    
    return await createTag(workflowId, {
      tag: newTag,
      message: message
    });
  }
}

// Usage
const versionManager = new VersionManager();

// Create patch release
await versionManager.createRelease('wf_abc123', 'patch', 'Bug fixes');
```

### 2. Automated Backup Strategy

```javascript
// Set up automated backup before critical changes
class BackupStrategy {
  constructor() {
    this.criticalOperations = ['deploy', 'restore', 'bulk_update'];
  }

  async beforeOperation(workflowId, operation) {
    if (this.criticalOperations.includes(operation)) {
      await this.createAutoBackup(workflowId, operation);
    }
  }

  async createAutoBackup(workflowId, reason) {
    const timestamp = new Date().toISOString().replace(/[:.]/g, '-');
    
    return await createTag(workflowId, {
      tag: `auto-backup-${timestamp}`,
      message: `Automatic backup before ${reason}`,
      type: 'backup'
    });
  }

  // Schedule regular backups
  scheduleRegularBackups(workflowId, interval = 'daily') {
    const schedules = {
      daily: 24 * 60 * 60 * 1000,
      weekly: 7 * 24 * 60 * 60 * 1000,
      monthly: 30 * 24 * 60 * 60 * 1000
    };

    const intervalMs = schedules[interval];
    if (!intervalMs) throw new Error('Invalid interval');

    setInterval(async () => {
      try {
        await this.createAutoBackup(workflowId, `scheduled_${interval}`);
        console.log(`${interval} backup created for workflow ${workflowId}`);
      } catch (error) {
        console.error('Backup failed:', error);
      }
    }, intervalMs);
  }
}
```

      html += this.generateChangeHtml(change);
    }
    
    html += '</div>';
    return html;
  }

  generateSummary(summary) {
    return `
      <div class="diff-summary">
        <h3>Changes Summary</h3>
        <ul>
          <li>Blocks: +${summary.blocks_added} -${summary.blocks_removed} ~${summary.blocks_modified}</li>
          <li>Connections: +${summary.connections_added} -${summary.connections_removed} ~${summary.connections_modified}</li>
        </ul>
      </div>
    `;
  }

  generateChangeHtml(change) {
    const icons = {
      block_added: '➕',
      block_removed: '➖',
      block_modified: '✏️',
      connection_added: '🔗',
      connection_removed: '💔',
      connection_modified: '🔄'
    };

    return `
      <div class="diff-change ${change.type}">
        <span class="change-icon">${icons[change.type] || '❓'}</span>
        <div class="change-details">
          <strong>${change.type.replace('_', ' ').toUpperCase()}</strong>
          <p>${this.formatChangeDescription(change)}</p>
        </div>
      </div>
    `;
  }

  formatChangeDescription(change) {
    switch (change.type) {
      case 'block_added':
        return `Added block "${change.block_name}" (${change.block_id})`;
      case 'block_removed':
        return `Removed block "${change.block_name}" (${change.block_id})`;
      case 'block_modified':
        const fieldChanges = change.changes.map(c => 
          `${c.field}: ${c.from} → ${c.to}`
        ).join(', ');
        return `Modified block "${change.block_name}": ${fieldChanges}`;
      default:
        return change.description || 'No description available';
    }
  }
}
