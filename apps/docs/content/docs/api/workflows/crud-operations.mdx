# Workflow CRUD Operations

This guide covers the fundamental Create, Read, Update, and Delete operations for workflows. These endpoints provide the core functionality for managing workflows in your Sim account.

## Create Workflow

Create a new workflow with basic metadata and an initial starter block.

### Endpoint

```http
POST /api/workflows
```

### Request Body

```json
{
  "name": "string (required)",
  "description": "string (optional)",
  "color": "string (optional, default: '#3972F6')",
  "workspaceId": "string (optional)",
  "folderId": "string | null (optional)",
  "templateId": "string (optional)",
  "cloneFromId": "string (optional)",
  "yaml": "string (optional)",
  "tags": ["string"] // (optional, default: [])
}
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `name` | string | **Required.** The workflow name (1-100 characters) |
| `description` | string | Optional description (max 500 characters) |
| `color` | string | Hex color code for workflow display (default: `#3972F6`) |
| `workspaceId` | string | ID of workspace to create workflow in |
| `folderId` | string\|null | ID of folder to organize workflow in |
| `templateId` | string | Create workflow from existing template |
| `cloneFromId` | string | Clone workflow from existing workflow |
| `yaml` | string | Create workflow from YAML definition |
| `tags` | string[] | Tags for workflow organization |

## Read Workflow

Retrieve a single workflow by ID, including its complete definition and metadata.

### Endpoint

```http
GET /api/workflows/{id}
```

### Path Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | string | **Required.** The workflow ID |

### Examples

<Tabs items={['JavaScript', 'cURL', 'Python']}>
<Tab value="JavaScript">
```javascript
// Get workflow by ID
const response = await fetch('/api/workflows/wf_abc123', {
  headers: {
    'x-api-key': 'YOUR_API_KEY'
  }
});

const { data: workflow } = await response.json();
console.log('Workflow name:', workflow.name);
console.log('Block count:', Object.keys(workflow.state.blocks).length);
console.log('Is deployed:', workflow.isDeployed);
```
</Tab>
<Tab value="cURL">

## Update Workflow

<Callout type="info">
**Note**: This endpoint only updates workflow metadata. To update the workflow structure (blocks, connections, etc.), use the collaborative editing endpoints or YAML import.
</Callout>

### Endpoint

```http
PUT /api/workflows/{id}
```

### Path Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | string | **Required.** The workflow ID |

### Request Body

```json
{
  "name": "string (optional)",
  "description": "string (optional)",
  "color": "string (optional)",
  "folderId": "string | null (optional)"
}
```

## Delete Workflow

Delete a workflow permanently. This action cannot be undone.

<Callout type="warn">
**Warning**: Deleting a workflow is permanent and cannot be undone. Consider creating a backup export before deletion.
</Callout>

### Endpoint

```http
DELETE /api/workflows/{id}
```

### Path Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | string | **Required.** The workflow ID |

### Query Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `check-templates` | boolean | Return template information before deletion |
| `deleteTemplates` | string | `delete` or `keep` - what to do with published templates |

### Examples

<Tabs items={['Simple Delete', 'Check Templates', 'Handle Templates']}>
<Tab value="Simple Delete">
```javascript
// Simple workflow deletion
const response = await fetch('/api/workflows/wf_abc123', {
  method: 'DELETE',
  headers: {
    'x-api-key': 'YOUR_API_KEY'
  }
});

if (response.ok) {
  console.log('Workflow deleted successfully');
} else {
  const error = await response.json();
  console.error('Delete failed:', error.message);
}
```
</Tab>
<Tab value="Check Templates">
```javascript
// Check if workflow has published templates before deletion
const response = await fetch('/api/workflows/wf_abc123?check-templates=true', {
  method: 'DELETE',
  headers: {
    'x-api-key': 'YOUR_API_KEY'
  }
});

const info = await response.json();
if (info.hasPublishedTemplates) {
  console.log(`This workflow has ${info.count} published templates:`);
  info.publishedTemplates.forEach(template => {
    console.log(`- ${template.name} (${template.views} views, ${template.stars} stars)`);
  });
  
  // User needs to decide what to do with templates
  const shouldDeleteTemplates = confirm('Delete templates as well?');
  // Proceed with actual deletion (see next example)
}
```
</Tab>
<Tab value="Handle Templates">
```javascript
// Delete workflow and handle templates
const deleteTemplates = 'delete'; // or 'keep'

const response = await fetch(`/api/workflows/wf_abc123?deleteTemplates=${deleteTemplates}`, {
  method: 'DELETE',
  headers: {
    'x-api-key': 'YOUR_API_KEY'
  }
});

if (response.ok) {
  if (deleteTemplates === 'delete') {
    console.log('Workflow and all templates deleted');
  } else {
    console.log('Workflow deleted, templates preserved as orphaned');
  }
}
```

### Error Handling Example

```javascript
async function createWorkflow(workflowData) {
  try {
    const response = await fetch('/api/workflows', {
      method: 'POST',
      headers: {
        'x-api-key': 'YOUR_API_KEY',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(workflowData)
    });

    if (!response.ok) {
      const error = await response.json();
      
      switch (response.status) {
        case 400:
          console.error('Invalid data:', error.details);
          // Handle validation errors
          break;
        case 403:
          console.error('Access denied:', error.message);
          // Handle permission errors
          break;
        case 409:
          console.error('Conflict:', error.message);
          // Handle naming conflicts
          break;
        default:
          console.error('Unexpected error:', error);
      }
      
      throw new Error(error.message);
    }

    return await response.json();
  } catch (error) {
    console.error('Request failed:', error);
    throw error;
  }
}
```

## Permissions

Different operations require different permission levels:

| Operation | Required Permission | Notes |
|-----------|-------------------|-------|
| **Create** | Workspace write access | Can create in owned workspaces or with write permission |
| **Read** | Workflow access | Owner, collaborator, or workspace member |
| **Update** | Workflow write access | Owner, editor collaborator, or workspace admin |
| **Delete** | Workflow admin access | Owner or workspace admin only |

### Permission Hierarchy

1. **Workflow Owner** - Full access to all operations
2. **Workspace Admin** - Full access within workspace workflows
3. **Workspace Editor** - Can create, read, update (not delete)
4. **Collaborator (admin)** - Can manage workflow and collaborators
5. **Collaborator (edit)** - Can read and update workflow
6. **Collaborator (view)** - Can only read workflow
7. **Workspace Member** - Basic access based on workspace role
