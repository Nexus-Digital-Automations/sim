# Advanced Filtering

The Sim API provides powerful filtering, searching, and sorting capabilities for workflow lists. You can combine multiple filters to find exactly the workflows you need, whether you're building dashboards, automation tools, or administrative interfaces.

## Overview

The workflow list endpoint (`GET /api/workflows`) supports:

- **Text Search** - Search across workflow names and descriptions
- **Status Filtering** - Filter by deployment status
- **Workspace & Folder Filtering** - Scope to specific locations
- **Date Range Filtering** - Filter by creation or modification dates
- **Collaboration Filtering** - Find workflows by collaborator
- **Sorting** - Order results by various criteria
- **Pagination** - Handle large result sets efficiently

## Basic Filtering

### Endpoint

```http
GET /api/workflows
```

### Query Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `page` | integer | `1` | Page number for pagination |
| `limit` | integer | `20` | Results per page (max 100) |
| `search` | string | - | Search in workflow names and descriptions |
| `workspaceId` | string | - | Filter by workspace ID |
| `folderId` | string\|null | - | Filter by folder ID (null for root) |
| `status` | enum | `all` | `deployed`, `draft`, or `all` |
| `sortBy` | enum | `updatedAt` | `name`, `createdAt`, `updatedAt`, `runCount` |
| `sortOrder` | enum | `desc` | `asc` or `desc` |

## Advanced Parameters

### Extended Query Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `tags` | string | Comma-separated tag names |
| `hasTemplates` | boolean | Filter workflows that have published templates |
| `isPublished` | boolean | Filter by publish status |
| `collaboratorId` | string | Find workflows where user is a collaborator |
| `createdAfter` | string | ISO 8601 date - workflows created after this date |
| `createdBefore` | string | ISO 8601 date - workflows created before this date |
| `updatedAfter` | string | ISO 8601 date - workflows updated after this date |
| `updatedBefore` | string | ISO 8601 date - workflows updated before this date |
| `minRunCount` | integer | Minimum number of executions |
| `maxRunCount` | integer | Maximum number of executions |

class WorkflowAnalytics {
  constructor(apiKey) {
    this.apiKey = apiKey;
    this.baseUrl = '/api/workflows';
  }

  async getDashboardData() {
    // Get all workflows for overview
    const allWorkflows = await this.fetch({
      limit: 100,
      sortBy: 'updatedAt'
    });

    // Get deployed workflows
    const deployedWorkflows = await this.fetch({
      status: 'deployed',
      limit: 50
    });

    // Get recently active workflows
    const oneWeekAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000).toISOString();
    const recentlyActive = await this.fetch({
      updatedAfter: oneWeekAgo,
      status: 'deployed',
      sortBy: 'runCount',
      sortOrder: 'desc'
    });

    // Get template workflows
    const templates = await this.fetch({
      hasTemplates: 'true',
      sortBy: 'runCount',
      sortOrder: 'desc'
    });

    return {
      total: allWorkflows.pagination.total,
      deployed: deployedWorkflows.pagination.total,
      recentlyActive: recentlyActive.data,
      topTemplates: templates.data.slice(0, 10)
    };
  }

  async fetch(params) {
    const response = await fetch(this.baseUrl + '?' + new URLSearchParams(params), {
      headers: { 'x-api-key': this.apiKey }
    });
    return response.json();
  }
}

// Usage
const analytics = new WorkflowAnalytics('YOUR_API_KEY');
const dashboardData = await analytics.getDashboardData();
console.log('Dashboard metrics:', dashboardData);

## Pagination Handling

### Response Structure

```json
{
  "data": [
    // Array of workflow objects
  ],
  "pagination": {
    "page": 2,
    "limit": 20,
    "total": 157,
    "totalPages": 8,
    "hasNextPage": true,
    "hasPrevPage": true
  },
  "filters": {
    "search": "email campaign",
    "status": "deployed",
    "sortBy": "updatedAt",
    "sortOrder": "desc"
  }
}
```

### Pagination Helper

```javascript
class WorkflowPaginator {
  constructor(apiKey, baseFilters = {}) {
    this.apiKey = apiKey;
    this.baseFilters = baseFilters;
    this.currentPage = 1;
    this.limit = 20;
  }

  async getPage(page = 1, additionalFilters = {}) {
    const params = {
      ...this.baseFilters,
      ...additionalFilters,
      page: page.toString(),
      limit: this.limit.toString()
    };

    const response = await fetch('/api/workflows?' + new URLSearchParams(params), {
      headers: { 'x-api-key': this.apiKey }
    });

    const result = await response.json();
    this.currentPage = page;
    return result;
  }

  async getAllPages(additionalFilters = {}) {
    const firstPage = await this.getPage(1, additionalFilters);
    const allWorkflows = [...firstPage.data];
    
    // Fetch remaining pages
    for (let page = 2; page <= firstPage.pagination.totalPages; page++) {
      const pageResult = await this.getPage(page, additionalFilters);
      allWorkflows.push(...pageResult.data);
    }
    
    return {
      data: allWorkflows,
      total: firstPage.pagination.total
    };
  }

  async *iteratePages(additionalFilters = {}) {
    let page = 1;
    let hasMore = true;

    while (hasMore) {
      const result = await this.getPage(page, additionalFilters);
      yield result;
      
      hasMore = result.pagination.hasNextPage;
      page++;
    }
  }
}

// Usage examples
const paginator = new WorkflowPaginator('YOUR_API_KEY', { 
  status: 'deployed' 
});

// Get specific page
const page2 = await paginator.getPage(2, { search: 'email' });

// Get all workflows (careful with large datasets)
const allDeployed = await paginator.getAllPages();

// Iterate through pages efficiently
for await (const page of paginator.iteratePages({ workspaceId: 'ws_123' })) {
  console.log(`Processing ${page.data.length} workflows from page ${page.pagination.page}`);
  // Process workflows in batches
}
```

// Process large numbers of workflows efficiently
async function processAllWorkflows(processor, batchSize = 50) {
  const paginator = new WorkflowPaginator('YOUR_API_KEY');
  paginator.limit = batchSize;
  
  for await (const page of paginator.iteratePages()) {
    console.log(`Processing batch of ${page.data.length} workflows...`);
    
    // Process workflows in parallel (but limit concurrency)
    const batchPromises = page.data.map(workflow => processor(workflow));
    await Promise.all(batchPromises);
    
    console.log(`Completed batch ${page.pagination.page}/${page.pagination.totalPages}`);
    
    // Add small delay to avoid overwhelming the API
    await new Promise(resolve => setTimeout(resolve, 100));
  }
}

## Common Filter Patterns

### Workspace Management

```javascript
// Get workspace overview
async function getWorkspaceOverview(workspaceId) {
  const [total, deployed, drafts, recent] = await Promise.all([
    fetch(`/api/workflows?workspaceId=${workspaceId}&limit=1`),
    fetch(`/api/workflows?workspaceId=${workspaceId}&status=deployed&limit=1`),
    fetch(`/api/workflows?workspaceId=${workspaceId}&status=draft&limit=1`),
    fetch(`/api/workflows?workspaceId=${workspaceId}&createdAfter=${new Date(Date.now() - 7*24*60*60*1000).toISOString()}&limit=10`)
  ].map(p => p.then(r => r.json())));

  return {
    totalWorkflows: total.pagination.total,
    deployedWorkflows: deployed.pagination.total,
    draftWorkflows: drafts.pagination.total,
    recentWorkflows: recent.data
  };
}
```

// Find workflows needing attention

}
